# AI Tavern Game - 项目总览

## 1. 项目愿景

AI Tavern Game 是一个高度可定制化的 AI 驱动角色扮演游戏，结合了：
- **ERA 风格文本 UI**：基于网格的地图、结构化状态显示、菜单驱动的交互，呈现文本艺术美学。
- **SillyTavern 风格 LLM 集成**：利用大语言模型生成丰富、动态的叙事文本、对话和事件描述。

目标是创建一个**游戏引擎**，内容创作者可以通过配置文件定义自己的世界、角色和场景，而 LLM 提供叙事层，让这些元素栩栩如生。

## 2. 技术栈

| 层级 | 技术 | 职责 |
|------|------|------|
| 前端 | React (TypeScript) | UI 渲染、用户交互、地图显示 |
| 后端 | Python (FastAPI) | 游戏状态管理、LLM 调度、游戏逻辑 |
| 通信 | WebSocket + REST API | 实时状态同步、动作请求 |
| LLM | 可配置（OpenAI / Claude / 本地模型） | 叙事生成、对话、事件描述 |
| 数据 | JSON 配置文件 | 游戏定义（地图、角色等）和存档状态 |

## 4. 核心系统

### 4.1 地图系统
> 详见: [map-system.md](./map-system.md)

- ERA 风格的网格地图显示
- 两种方格类型：**可移动方格**（带编号的位置）和**装饰性方格**（仅用于视觉表现）
- 多地图标签页组织，通过方格连接实现跨地图导航
- 单个地图标签页内可包含多个子地图（如同一标签页中的"2F"和"3F"）
- 玩家位置显示在可移动方格上（绿色高亮）
- 完全通过 JSON 配置文件定义

### 4.2 角色系统
> 详见: [character-system.md](./character-system.md)

- 玩家角色和 NPC，可自定义属性
- 角色在地图上的定位（仅限可移动方格）
- 角色状态、关系和物品栏
- 角色在地图中的位置追踪
- 通过 LLM 驱动 NPC 行为和对话

### 4.3 LLM 集成
> 详见: [llm-config.md](./llm-config.md)

- 可配置的 LLM 提供商和模型
- 不同游戏场景的系统提示词模板（探索、对话、战斗等）
- 上下文管理：将相关游戏状态输入 LLM
- 结构化输出解析，用于游戏机制效果
- 流式响应显示

### 4.4 游戏状态管理

所有游戏状态存在于**后端**，前端是纯渲染层。

核心状态包括：
- 当前地图和玩家位置
- 所有角色状态（位置、属性、物品栏、关系）
- 游戏时钟 / 时间系统
- 事件标志和任务进度
- 对话历史（用于 LLM 上下文）

### 4.5 动作系统

玩家通过**菜单驱动的动作系统**进行交互：
- **移动**：点击连接的方格编号或从移动选项中选择
- **交互**：基于位置和附近角色的上下文敏感动作
- **物品/状态**：查看和管理角色状态
- **对话**：与 NPC 发起对话（触发 LLM 生成）

## 5. 定制化理念

本项目设计为**引擎**而非单一游戏。内容创作者可以：
1. 通过 JSON 配置文件定义地图（网格布局、方格属性、连接关系）
2. 创建自定义属性、背景故事和行为提示词的角色
3. 按场景配置 LLM 行为（系统提示词、温度、响应格式）
4. 通过配置系统定义事件、触发器和游戏逻辑
