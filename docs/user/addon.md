# 扩展包 (Add-on) 用户指南

## 什么是扩展包？

扩展包是游戏内容的模块化单元。游戏中的所有实体定义（角色、地图、特质、服装、物品、行动、变量、事件等）都存放在扩展包中，而非世界目录本身。世界配置仅记录「启用了哪些扩展包的哪个版本」。

一个扩展包可以包含以下任意组合的内容：

| 类型 | 说明 |
|------|------|
| 特质 (traits) | 角色的性格、种族、能力等 |
| 特质分组 (trait groups) | 将特质归类为互斥组 |
| 服装 (clothing) | 可穿戴的服装定义 |
| 物品 (items) | 可拾取/使用的物品 |
| 行动 (actions) | 角色可执行的行动及其结果 |
| 角色 (characters) | NPC 和玩家角色配置 |
| 地图 (maps) | 游戏地图和格子定义 |
| 变量 (variables) | 派生计算变量 |
| 事件 (events) | 全局事件定义 |
| 世界变量 (world variables) | 全局状态变量 |
| 装饰预设 (decor presets) | 地图装饰符号预设 |

---

## 扩展包类型

### 独立包 (Independent)
自包含的扩展包，不依赖其他扩展包即可运行。例如一个独立的「基础包」提供种族、特质、物品等基础定义。

### 补充包 (Supplementary)
依赖其他扩展包的内容。例如一个「东方角色包」可能依赖「基础包」中定义的特质和物品。依赖关系在保存时会被自动检测并写入 addon.json。

---

## 启用和禁用扩展包

1. 点击导航栏右侧的 **[扩展]** 按钮，打开扩展包侧边栏
2. 侧边栏列出所有已安装的扩展包，每个扩展包旁有一个开关
3. **打开开关** = 将该扩展包加入当前世界的加载列表
4. **关闭开关** = 从加载列表中移除
5. 修改后，点击底部的 **[保存变更]** 按钮，变更才会生效

> **加载顺序很重要**：列表中靠后的扩展包会覆盖靠前扩展包中同 ID 的实体定义。如果两个扩展包定义了同一个 ID 的特质，后加载的版本生效。

> **注意**：禁用一个扩展包不会删除它的文件，只是不再加载它的内容。

## 本体与分支 (Fork)

### 什么是本体版本？

本体是扩展包的原始版本（如 `1.0.0`），由扩展包作者发布。多个世界可以共享同一个本体版本——但这意味着在一个世界中做的修改会影响所有使用该版本的世界。

### 什么是分支版本？

分支是本体的世界专属副本。版本号格式为 `{基础版本}-{世界ID}`，例如 `1.0.0-myworld`。

分支版本中的修改只影响该世界，不会波及其他世界或本体。

### 何时使用本体？何时创建分支？

| 场景 | 建议 |
|------|------|
| 只有一个世界使用该扩展包 | 直接使用本体即可 |
| 多个世界共享同一套定义 | 使用本体，所有世界同步更新 |
| 想在某个世界中做独立修改 | 创建分支 |
| 想试验性修改但不影响原版 | 创建分支 |

### 如何创建分支

1. 在扩展包侧边栏中点击某个扩展包，展开版本面板
2. 点击 **[为当前世界创建分支]** 按钮
3. 系统会复制当前版本的所有内容到一个新的分支目录
4. 当前世界自动切换到分支版本

## 版本管理

### 切换版本

1. 展开某个扩展包的版本面板
2. 在版本列表中点击想要切换到的版本
3. 点击 **[保存变更]** 使切换生效

版本列表中会显示每个版本的类型：
- 无标签 = 本体版本
- `分支: {世界名}` = 某个世界的分支版本

### 创建新版本 (复制)

在版本管理面板中，可以将当前版本复制到一个新的版本号：

1. 点击 **[新建版本]**
2. 输入新版本号（如 `1.1.0`）
3. 系统会完整复制当前版本的所有实体文件到新版本目录

这适用于：
- 版本升级（从 1.0.0 创建 1.1.0）
- 创建实验分支

### 覆盖版本

可以用一个版本的实体内容覆盖另一个版本：

1. 在版本管理面板中选择源版本和目标版本
2. 点击 **[覆盖]**
3. 目标版本的所有实体文件被替换为源版本的内容
4. 目标版本的 addon.json（版本号、分支标记等）保持不变

这适用于将分支上的修改合并回本体。

## 编辑扩展包元信息

点击扩展包卡片右上角的齿轮图标，可以编辑以下信息：

| 字段 | 说明 |
|------|------|
| 名称 (name) | 扩展包显示名称 |
| 作者 (author) | 扩展包作者 |
| 描述 (description) | 扩展包简介 |
| 封面 (cover) | 封面图片（支持上传 png/jpg/gif/webp） |

> 这些元信息是**跨版本共享**的，存储在 `about/meta.json` 中。修改后所有版本都会显示相同的名称、描述等。

---

## 创建新扩展包

1. 在扩展包侧边栏底部，点击 **[新建扩展包]** 按钮
2. 填写以下信息：
   - **ID**：唯一标识符（英文、数字、连字符），创建后不可更改
   - **名称**：显示名称
   - **版本**：初始版本号（默认 `1.0.0`）
   - **作者**：（可选）
   - **描述**：（可选）
3. 点击确认，系统会创建空的扩展包目录结构
4. 之后可以在编辑器中向该扩展包添加角色、地图、行动等内容

---

## 删除扩展包

### 删除单个版本
在版本管理面板中，点击某个版本旁的删除按钮。

> 不能删除当前世界正在使用的版本。请先禁用该扩展包或切换到其他版本。

### 删除整个扩展包
在扩展包卡片上点击删除按钮，会删除该扩展包的所有版本和元信息。

> 不能删除当前世界正在使用的扩展包。请先在侧边栏中禁用它。

## 典型使用场景

### 场景 1：全新开始
1. 创建一个新扩展包 `my-game`
2. 创建一个新世界，启用 `my-game`
3. 在编辑器中添加地图、角色、行动等
4. 保存变更

### 场景 2：基于现有扩展包搭建
1. 启用已有的基础扩展包（如 `base`）
2. 创建自己的补充包（如 `my-characters`），添加自定义角色和行动
3. 两个扩展包同时启用，自定义包中可以引用基础包的特质和物品

### 场景 3：多世界并行
1. 一个本体扩展包 `era-koumakan@1.0.0` 被多个世界共享
2. 世界 A 直接使用本体 `1.0.0`
3. 世界 B 创建分支 `1.0.0-worldB`，在分支上做独立修改
4. 本体的更新不会影响世界 B 的分支

### 场景 4：合并分支修改回本体
1. 在分支 `1.0.0-myworld` 上完成修改和测试
2. 使用版本管理的「覆盖」功能，将分支内容写回本体 `1.0.0`
3. 所有使用本体的世界都会获得更新
