# 行动（Action）

## 概述

**行动是角色在游戏中可以执行的操作。** 例如「交谈」「采集草药」「攻击」「休息」等。每个行动定义了：

- 什么条件下可以执行
- 执行需要消耗什么
- 执行后会产生什么结果

行动不仅是玩家的操作选项，NPC 也会根据权重系统自动选择行动来执行。

**编辑入口**：导航栏 → 行动编辑页

## 基础属性

| 字段 | 说明 |
|------|------|
| **ID** | 行动的唯一标识符，在扩展包内不可重复 |
| **name** | 行动的显示名称，如「友好交谈」 |
| **category** | 分类标签，如 `social`、`combat`、`gather`，用于分组和行动链引用 |
| **targetType** | 行动的目标类型（见下表） |
| **timeCost** | 执行该行动消耗的游戏时间（单位：分钟，须为 5 的倍数） |
| **triggerLLM** | 是否在执行时触发 LLM 生成文本描述 |

### 目标类型（targetType）

| 值 | 说明 |
|------|------|
| `none` | 无目标，行动只影响自身（如「休息」「冥想」） |
| `npc` | 需要选择一个 NPC 作为目标（如「交谈」「攻击」） |
| `self` | 目标是自己（如「自我治疗」） |

## 条件系统

行动可以设置执行条件，只有满足所有条件时行动才会出现在可用列表中。

### 条件类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `location` | 必须在指定格子中 | 角色位于「酒馆大厅」 |
| `npcPresent` | 指定 NPC 必须在同一格子 | 酒保在场 |
| `npcAbsent` | 指定 NPC 不能在同一格子 | 守卫不在场 |
| `resource` | 资源值满足条件 | 体力 ≥ 20 |
| `ability` | 能力等级满足条件 | 剑术 ≥ 3 |
| `trait` | 拥有指定特质 | 拥有「人类」特质 |
| `noTrait` | 没有指定特质 | 没有「失明」特质 |
| `favorability` | 与目标的好感度满足条件 | 好感 ≥ 50 |
| `hasItem` | 持有指定物品 | 背包里有「铁矿石」 |
| `clothing` | 服装槽位状态或指定衣物 | 上衣槽穿着「铁甲」、饰品槽非空 |
| `time` | 当前游戏时间满足条件 | 18:00～06:00、星期六、冬季 |
| `basicInfo` | 基础信息满足条件 | 年龄 ≥ 18 |
| `variable` | 衍生变量满足条件 | 战斗力 ≥ 100 |
| `worldVar` | 世界变量满足条件 | 剧情阶段 = 2 |

### 比较运算符

| 运算符 | 说明 |
|--------|------|
| `>=` | 大于等于 |
| `<=` | 小于等于 |
| `>` | 大于 |
| `<` | 小于 |
| `==` | 等于 |
| `!=` | 不等于 |

### 条件目标（condTarget）

对于涉及角色属性的条件（如 resource、ability、trait 等），需要指定检查谁的属性：

| 值 | 说明 |
|------|------|
| `self` | 检查执行者自身的属性 |
| `target` | 检查行动目标的属性 |

### 条件组合

条件支持逻辑组合，实现复杂判断：

- **AND**：所有子条件都满足才通过（默认行为）
- **OR**：任一子条件满足即通过
- **NOT**：子条件不满足才通过

可以嵌套使用，例如：`AND( 体力≥20, OR( 拥有剑术, 拥有魔法 ) )` 表示「体力足够，且会剑术或魔法」。

## 消耗

行动可以设定执行时扣除的消耗，执行后立即生效。

| 消耗类型 | 说明 | 示例 |
|----------|------|------|
| `resource` | 扣除资源值 | 消耗 10 点体力 |
| `basicInfo` | 扣除基础信息数值 | 消耗 5 金币 |
| `item` | 扣除物品数量 | 消耗 1 个「铁矿石」 |

## 结果（Outcome）

一个行动可以有多个**结果**，每次执行时根据权重随机选择一个。结果代表行动的不同可能走向。

### 结果属性

| 字段 | 说明 |
|------|------|
| **grade** | 结果等级，如 `success`、`failure`、`critical` 等，纯标签用途 |
| **label** | 结果的显示标签，如「大成功」「失败」 |
| **weight** | 基础权重，数值越大越容易被选中 |
| **weightModifiers** | 权重修正器列表，根据条件动态调整该结果的被选概率（详见「值修正器」章节） |
| **effects** | 该结果触发的效果列表（详见「效果系统」章节） |

### 权重机制

假设一个行动有三个结果：

| 结果 | 权重 |
|------|------|
| 大成功 | 10 |
| 普通成功 | 60 |
| 失败 | 30 |

则大成功的概率为 10%，普通成功为 60%，失败为 30%。权重修正器可以在运行时动态调整这些权重。

## 效果系统

效果定义了结果触发后对游戏状态的具体改变。每个结果可以包含多个效果。

### 效果类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `resource` | 修改资源值 | 恢复 20 点生命 |
| `ability` | 修改能力经验值 | 剑术经验 +50 |
| `basicInfo` | 修改基础信息 | 金币 +50 |
| `favorability` | 修改好感度（通过 favFrom/favTo 指定谁对谁） | 目标对执行者好感 +10 |
| `trait` | 添加或移除特质 | 获得「中毒」特质 |
| `item` | 添加或移除物品 | 获得「药草」×3 |
| `clothing` | 穿上或脱下服装 | 穿上「皮甲」 |
| `position` | 改变角色位置 | 移动到「地牢入口」 |
| `experience` | 记录角色经历次数 | 记录一次「被攻击」经历 |
| `worldVar` | 修改世界变量 | 剧情阶段 +1 |

### 操作方式（op）

| op | 适用类型 | 说明 |
|------|----------|------|
| `add` | resource / ability / basicInfo / favorability / item / trait / experience / worldVar | 增减数值，或添加物品/特质 |
| `set` | resource / ability / basicInfo / favorability / clothing / worldVar | 直接设置为指定值/状态 |
| `remove` | item / trait / clothing | 移除物品/特质，或脱下服装 |

### 效果目标（target）

| 值 | 说明 |
|------|------|
| `self` | 效果作用于行动执行者 |
| `{{targetId}}` | 效果作用于行动目标（targetType 为 npc 时有效） |
| 具体角色 ID | 效果作用于指定角色 |

### 好感度方向（favFrom / favTo）

`favorability` 类型的效果通过 `favFrom` 和 `favTo` 指定「谁对谁」的好感度变化：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| **favFrom** | 谁的好感度发生变化 | `{{targetId}}`（行动目标） |
| **favTo** | 对谁的好感度 | `self`（行动执行者） |

可用的值：`self`（执行者）、`{{targetId}}`（行动目标）、`{{player}}`（玩家角色）、具体角色 ID。

默认行为是「目标对执行者的好感度变化」。例如玩家对 NPC 执行「友好交谈」，默认效果是 NPC 对玩家的好感 +5。

### 百分比模式（valuePercent）

当 `valuePercent` 为 `true` 时，效果值会被解释为百分比。例如 `value: 20` + `valuePercent: true` 表示增加当前值的 20%，而非固定增加 20。

### 变量引用值

效果的 `value` 除了直接填数字，还可以引用衍生变量：

```
value: { varId: "战斗力", multiply: 0.5 }
```

实际值 = 变量「战斗力」的计算结果 × 0.5。适合做动态伤害、动态回复等。

### 值修正器（valueModifiers）

效果的数值可以被修正器动态调整。例如「治疗」行动的恢复量可以根据医术等级增加。详见下方「值修正器」章节。

## 值修正器（ValueModifier）

值修正器用于动态调整数值（结果权重、效果数值等）。它根据角色的属性来计算加成。

### 修正器属性

| 字段 | 说明 |
|------|------|
| **type** | 参照的属性类型（见下表） |
| **id** | 参照的具体属性 ID（如某个能力的 ID） |
| **bonus** | 每单位提供的加成值 |
| **bonusMode** | 加成模式：`add`（固定值累加，默认）或 `multiply`（百分比乘数） |
| **per** | 每多少单位的属性值提供一次加成 |

### 参照类型（type）

| 类型 | 说明 | 示例 |
|------|------|------|
| `ability` | 参照能力等级 | 每 1 级剑术，伤害 +5 |
| `trait` | 参照是否拥有特质 | 拥有「强壮」时，加成 +10 |
| `favorability` | 参照好感度 | 每 10 点好感，成功率 +5% |
| `experience` | 参照能力经验值 | 每 100 经验，效果 +1 |
| `variable` | 参照衍生变量 | 每 1 点战斗力，伤害 +2 |
| `worldVar` | 参照世界变量 | 难度每增加 1，消耗 +10% |

### 计算示例

假设一个治疗效果的配置：
- 基础恢复：20 点生命
- 值修正器：type=`ability`, id=`医术`, bonus=`5`, per=`1`

如果执行者的医术等级为 3，则实际恢复 = 20 + (3 ÷ 1) × 5 = **35 点生命**。

## 行动链（suggestNext）

行动链用于让某些行动在执行后增加特定后续行动的选中概率，形成自然的行为序列。

### 属性

| 字段 | 说明 |
|------|------|
| **actionId** | 推荐的后续行动 ID（与 category 二选一） |
| **category** | 推荐的后续行动分类（与 actionId 二选一） |
| **bonus** | 权重加成值，叠加到后续行动的 NPC 权重上 |
| **decay** | 衰减时间（游戏分钟，须为 5 的倍数），经过此时间后加成归零 |

### 衰减机制

行动链的加成会随时间线性衰减。刚执行完时加成最高，随着游戏时间推移逐渐减弱，经过 decay 分钟后完全消失。这避免了 NPC 无限重复同一行为模式。

### 示例

「友好交谈」的 suggestNext 配置：
- actionId: `赠送礼物`, bonus: `20`
- category: `social`, bonus: `10`

NPC 执行完「友好交谈」后，接下来选择「赠送礼物」的概率会暂时提高 20 点权重，选择其他社交类行动的概率提高 10 点权重。

## 输出模板

行动执行后显示的文本内容。

### 基础模板（outputTemplate）

直接定义一段固定文本模板，每次执行该结果时都使用它。

### 条件模板（outputTemplates）

定义多个模板，根据条件选择使用哪一个。适用于需要根据情境显示不同描述的场景。

### 模板变量

模板中可以使用 `{{变量}}` 插入动态内容：

**全局变量**

| 变量 | 说明 |
|------|------|
| `{{player}}` / `{{self}}` | 执行者角色的名称 |
| `{{target}}` | 行动目标的名称 |
| `{{outcome}}` | 结果的标签（如「大成功」） |
| `{{outcomeGrade}}` | 结果的等级标识（如 `success`） |
| `{{effects}}` | 效果摘要（逗号分隔） |
| `{{time}}` | 当前游戏时间 |
| `{{weather}}` | 当前天气 |
| `{{location}}` | 执行者所在格子名称 |

**角色属性变量**（`self` = 执行者，`target` = 目标，格式相同）

| 变量 | 说明 |
|------|------|
| `{{self.resource.X}}` | 资源 X 的当前值 |
| `{{self.ability.X}}` | 能力 X 的等级 |
| `{{self.abilityExp.X}}` | 能力 X 的经验值 |
| `{{self.basicInfo.X}}` | 基础信息 X 的值（如 `money`） |
| `{{self.clothing.X}}` | 槽位 X 的服装名称（未穿着显示「无」） |
| `{{self.trait.X}}` | 特质分类 X 的值（逗号分隔） |
| `{{self.favorability.角色ID}}` | 对指定角色的好感度 |
| `{{self.inventory.物品ID}}` | 持有指定物品的数量（无则 0） |
| `{{self.experience.X}}` | 经验记录 X 的次数（无则 0） |

## NPC 权重

NPC 通过权重系统自动决定执行哪个行动。

### 属性

| 字段 | 说明 |
|------|------|
| **npcWeight** | 行动的基础 NPC 权重，数值越大 NPC 越倾向选择该行动 |
| **npcWeightModifiers** | NPC 权重修正器列表，结构与值修正器相同 |

### NPC 决策流程

1. **扫描可达格子**：NPC 从当前位置出发，检查可达范围内的所有格子
2. **查找可用行动**：对每个格子，查找该位置可用的行动
3. **感知过滤**：只考虑感知范围内能"看到"的目标（受地图连接的感知阻断影响）
4. **评估条件**：对每个行动检查条件是否满足
5. **计算权重**：基础权重 + 权重修正器加成 + 行动链加成
6. **随机选择**：按最终权重进行加权随机，选出要执行的行动

权重为 0 或负数的行动不会被选择。

## 典型用例

以下是一个社交行动「友好交谈」的完整配置示例：

**基础属性**

| 字段 | 值 |
|------|------|
| ID | `friendly_talk` |
| name | 友好交谈 |
| category | `social` |
| targetType | `npc` |
| timeCost | 30（分钟） |

**条件**

| 条件 | 说明 |
|------|------|
| location: `tavern_hall` | 必须在酒馆大厅 |
| resource: 体力 ≥ 10, condTarget: `self` | 自身体力不低于 10 |
| noTrait: `mute`, condTarget: `self` | 自身没有「哑巴」特质 |

**消耗**

| 类型 | 说明 |
|------|------|
| resource: 体力 -5 | 消耗 5 点体力 |

**结果 1：愉快交谈（weight: 60）**

效果：
- favorability +5（目标好感度提升）
- experience: 口才经验 +10（执行者获得口才经验）

输出模板：`{{player}} 和 {{target}} 进行了一次愉快的交谈。`

**结果 2：话不投机（weight: 30）**

效果：
- favorability -2（目标好感度微降）

权重修正器：trait=`社交恐惧`, bonus=`20`（如果执行者有「社交恐惧」特质，此结果更容易出现）

**结果 3：深入交心（weight: 10）**

效果：
- favorability +15（目标好感度大幅提升）
- trait: 获得「挚友」特质

权重修正器：favorability, bonus=`2`, per=`10`（好感度每高 10 点，此结果权重 +2）

**行动链（suggestNext）**
- actionId: `give_gift`, bonus: `15`（交谈后更可能赠送礼物）
- category: `social`, bonus: `5`（交谈后更可能继续社交）

**NPC 权重**
- npcWeight: `40`
- npcWeightModifiers: favorability, bonus=`1`, per=`10`（好感度越高，NPC 越倾向发起交谈）
