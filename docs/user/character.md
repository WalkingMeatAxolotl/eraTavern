# 角色系统

角色是游戏世界中的核心实体。每个角色拥有基本信息、资源、特质、能力、装备、物品栏、好感度、经验等属性，并被放置在地图上的某个区域中。

---

## 什么是角色

角色代表游戏世界中的人物，包括玩家操控的主角和由系统自动行动的 NPC。所有角色共享同一套属性模板（character_template），模板定义了哪些字段存在、默认值是什么。

角色数据存储在扩展包的 `characters/` 目录下，每个角色一个 JSON 文件。

---

## 创建与删除角色

### 创建角色
在角色编辑器中点击 **[新建角色]**，填写角色 ID 和各项属性后点击 **[确定]** 即可创建。角色 ID 创建后不可修改。

新角色默认会归属到当前激活的扩展包下，ID 格式为 `扩展包ID.角色本地ID`。

### 删除角色
在编辑角色时点击 **[删除]** 按钮，确认后角色将从内存中移除。所有其他角色对该角色的好感度引用也会被自动清理。

> 注意：创建和删除操作均在内存中进行，需要通过 **[保存变更]** 才会写入磁盘。

---

## 基本信息（basicInfo）

基本信息由角色模板定义，包含若干自定义字段。每个字段有一个名称（label）和类型（string 或 number）。

常见字段示例：
- **名称** — 角色的显示名称（文本类型）
- **金钱** — 角色持有的金币数量（数值类型）

具体有哪些字段完全取决于当前使用的角色模板。

---

## 资源（Resources）

资源是带有当前值和最大值的数值属性，通常用于表示体力、气力等。每种资源还有一个显示颜色。

编辑器中显示为 `当前值 / 最大值` 的形式，可以分别修改。

资源的最大值可以被特质效果和装备效果修改。当最大值被修改时，当前值会自动限制为不超过最大值。

---

## 立绘（Portrait）

每个角色可以上传一张立绘图片。在编辑器中：
- 点击 **[选择图片]** 上传新立绘
- 点击 **[清除]** 移除当前立绘

立绘文件会被上传到扩展包的 `characters/` 资源目录中，角色数据中仅存储文件名。

---

## 位置（Position）

每个角色有两个位置属性：

### 初始位置
角色当前所在的地图和区域。由地图 ID（mapId）和区域 ID（cellId）组成。

### 休息位置
角色的默认休息地点。NPC 在没有其他目标时会返回此位置。

在编辑器中，先选择地图，再从该地图的区域列表中选择具体区域。

---

## 特质（Traits）

特质按类别分组（如族群、身体特征、精神特征等），每个类别可以持有多个特质。

特质的种类由扩展包中的特质定义决定。部分特质会归属到「特质组」中，特质组可以设置为排他（exclusive）——同一排他组中同时只能拥有一个特质。

### 特质效果
特质可以附带效果，自动修改角色的资源上限、能力值或基本信息中的数值字段。效果分为：
- **固定值**（fixed）：直接加减
- **百分比**（percentage）：按比例缩放

> 注意：`ability` 和 `experience` 类别的特质不在特质区显示，它们分别作为能力和经验单独展示。

---

## 能力（Abilities）

能力由特质定义中 `category: "ability"` 的条目自动生成，所有角色共享同一套能力列表。

每项能力有一个经验值（exp），系统自动根据经验值计算等级：
| 经验值 | 等级 |
|--------|------|
| 0-999 | G |
| 1000-1999 | F |
| 2000-2999 | E |
| 3000-3999 | D |
| 4000-4999 | C |
| 5000-5999 | B |
| 6000-6999 | A |
| 7000+ | S |

### 能力衰减
能力可以配置自动衰减（decay），按固定值或百分比在指定时间间隔后减少经验值。

---

## 经验（Experiences）

经验由特质定义中 `category: "experience"` 的条目自动生成。每项经验记录：
- **次数**（count）：发生了多少次
- **初次记录**（first）：首次发生时的事件、地点和对象

在编辑器中，可以手动设置初始次数。当次数设为 0 时，初次记录会被清除。

---

## 装备/服装（Clothing）

角色有若干装备槽位，由模板定义。默认槽位包括：

| 槽位 | 名称 |
|------|------|
| hat | 帽子 |
| upperBody | 上半身 |
| upperUnderwear | 上半身内衣 |
| lowerBody | 下半身 |
| lowerUnderwear | 下半身内衣 |
| hands | 手 |
| feet | 脚 |
| shoes | 鞋子 |
| accessory1 | 装饰品1 |
| accessory2 | 装饰品2 |
| accessory3 | 装饰品3 |

每个槽位可以装备一件服装，装备状态为 `worn`（穿着）或 `halfWorn`（半穿）。

### 遮挡机制（Occlusion）
服装可以定义遮挡列表——当服装处于 `worn` 状态时，会遮挡其他指定槽位。`halfWorn` 状态不会产生遮挡。

### 装备效果
与特质类似，服装也可以附带效果，修改角色的资源上限、能力值等。

---

## 物品栏（Inventory）

角色可以持有若干物品，每个物品条目包含物品 ID 和数量。

在编辑器中：
- 点击 **[+ 添加物品]** 从物品定义列表中选择
- 每个物品可以设置数量（不超过该物品的最大堆叠数）
- 点击 **x** 移除物品

---

## 好感度（Favorability）

好感度表示角色与其他角色之间的关系数值。每个角色可以对其他角色设置不同的好感度值（整数）。

在编辑器中：
- 从下拉列表选择目标角色添加好感度条目
- 设置数值（默认为 0）
- 点击 **x** 移除条目

未设置的角色间好感度默认为 0。

---

## 玩家角色（Player Character）

一个世界中只能有一个玩家角色。通过 PATCH 接口设置 `isPlayer: true` 来指定。当设置某角色为玩家角色时，其他角色的 `isPlayer` 会自动清除。

玩家角色由玩家手动操控行动，而非玩家角色（NPC）由系统 AI 自动决策行动。
