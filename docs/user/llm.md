# LLM 叙事增强

## 概述

LLM（大语言模型）叙事增强功能可以将游戏中模板生成的基础文本重写为更丰富、更沉浸的叙事描述。

**核心特点**：
- LLM 只负责**美化文本**，不参与任何游戏逻辑（不会改变属性、移动、物品等）
- 游戏输出区同时展示原始输出和 LLM 叙事，两者并列显示
- 支持任何 OpenAI 兼容的 API（本地或云端）

**编辑入口**：导航栏 → [LLM]

## 预设（Preset）

### 什么是预设

预设是一套完整的 LLM 调用配置，包含三部分：
- **API 连接信息**：服务器地址、密钥、模型选择
- **生成参数**：温度、最大 token 数等
- **提示词条目**：发送给 LLM 的指令和上下文

预设独立于扩展包和世界存储，保存在 `llm-presets/` 目录下。

### 创建和编辑预设

1. 点击导航栏 [LLM] 进入预设管理页
2. 点击 **[+ 新建预设]** 创建新预设
3. 填写基本信息：
   - **ID**：预设的唯一标识（创建后不可更改）
   - **名称**：显示名称
   - **描述**：说明这个预设的风格或用途

### API 配置

| 字段 | 说明 |
|------|------|
| **API URL** | LLM 服务的地址，如 `http://127.0.0.1:8080/v1` |
| **API Key** | 认证密钥（部分本地服务不需要，可留空） |
| **模型** | 要使用的模型名称。可以手动输入，也可以点击 [获取模型] 从服务器拉取可用列表 |
| **流式输出** | 开启后，LLM 生成的文本会逐字显示（打字效果）；关闭则等待全部生成后一次显示 |
| **后处理** | 「合并相邻同角色」会将连续的 system/user 消息合并，适配部分 API 限制 |

**[获取模型]**：填写 URL 和 Key 后点击，会从服务器获取可用模型列表，填入下拉框供选择。

**[测试连接]**：发送一个最小请求来验证 URL + Key + 模型是否可用。

### 生成参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **Temperature** | 0.8 | 控制输出的随机性。越高越多样，越低越确定 |
| **Max Tokens** | 4096 | 单次生成的最大 token 数 |
| **Top P** | 1.0 | 核采样参数 |
| **Freq Penalty** | 0 | 频率惩罚，减少重复用词 |
| **Pres Penalty** | 0 | 存在惩罚，鼓励新话题 |

### 提示词条目

提示词条目是发送给 LLM 的消息列表。每个条目有：

| 字段 | 说明 |
|------|------|
| **名称** | 条目的标签，便于识别 |
| **角色** | `system`（系统指令）、`user`（用户输入）、`assistant`（AI 回复预设） |
| **内容** | 提示词文本，支持变量插值 |
| **启用** | 可以关闭某个条目而不删除它 |

条目按位置排序（用 ▲▼ 箭头调整顺序），组合成发送给 LLM 的完整消息链。

### 可用变量

在提示词内容中使用 `{{变量名}}` 可以插入动态的游戏数据。编辑时可以在变量面板中点击直接插入。

| 变量 | 说明 |
|------|------|
| `{{rawOutput}}` | 本回合所有可见输出（行动结果 + NPC 行为）——这是 LLM 重写的核心输入 |
| `{{playerName}}` | 玩家角色名称 |
| `{{playerInfo}}` | 玩家完整状态（资源、特征、能力、物品、好感度） |
| `{{targetName}}` | 行动目标角色的名称（如有） |
| `{{targetInfo}}` | 行动目标的完整状态（如有） |
| `{{clothingState}}` | 玩家当前穿着的服装 |
| `{{location}}` | 当前所在位置名称 |
| `{{mapName}}` | 当前地图名称 |
| `{{time}}` | 游戏内时间（如「春3日 14:30」） |
| `{{weather}}` | 当前天气 |

### 示例预设

一个简单的叙事增强预设可能包含以下条目：

1. **系统指令**（system）：「你是一个角色扮演游戏的叙事 AI，负责将基础文本重写为沉浸式描述。」
2. **游戏状态**（system）：「当前位置：{{location}}，时间：{{time}} {{weather}}，玩家：{{playerName}}」
3. **原始输出**（user）：「以下是本回合发生的事情，请重写为叙事文本：\n\n{{rawOutput}}」

## 触发方式

### 自动触发

在行动编辑器中勾选「触发 LLM」后，玩家每次执行该行动都会自动调用 LLM 生成叙事文本。

### 手动触发

无论行动是否标记了自动触发，执行后输出区域都会显示 **[LLM 生成]** 按钮。点击即可手动触发 LLM 重写。

### 生成完成后

生成完成后，LLM 叙事区会显示完整文本和 **[重新生成]** 按钮。如果对结果不满意，可以点击重新生成。

如果生成出错（连接失败、认证错误等），会显示错误信息和 **[重试]** 按钮。

## 默认预设设置

预设的选择有三个层级，优先级从高到低：

| 层级 | 设置位置 | 说明 |
|------|---------|------|
| **行动级** | 行动编辑器 → 触发 LLM → LLM 预设 | 特定行动使用特定预设 |
| **世界级** | 设置页 → LLM 设置 → 世界级预设 | 当前世界的默认预设，覆盖全局 |
| **全局** | 设置页 → LLM 设置 → 全局默认预设 | 所有世界通用的默认 |

如果三个层级都没有设置预设，LLM 功能不会被自动触发（但手动点击 [LLM 生成] 也无法使用）。

## 注意事项

- LLM 功能需要有可用的 OpenAI 兼容 API 服务（本地如 Ollama、LM Studio，或云端如 OpenAI API）
- API 密钥保存在本地预设文件中，不会上传到任何地方
- 预设文件位于 `llm-presets/` 目录，该目录默认被 `.gitignore` 排除
- LLM 生成不影响游戏逻辑——属性变化、物品增减等都由游戏引擎处理，LLM 只是改善文本展示
