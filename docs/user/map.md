# 地图

## 概述

地图是游戏世界的空间系统。每张地图由一个**网格**组成，网格中的**区格（Cell）**代表具体的场所（房间、广场、道路等）。区格之间通过**连接**关联，定义了角色可以移动的路径。

地图编辑器提供可视化的网格编辑工具，你可以在其中绘制区格、设置连接、应用装饰预设，并为场所配置背景图。

**操作入口**：点击顶部导航栏的 **地图** 进入地图编辑页。

## 地图属性

选中一张地图后，可以在右侧面板编辑以下属性：

| 属性 | 说明 |
|------|------|
| **name** | 地图的显示名称 |
| **description** | 地图简介（可选） |
| **defaultColor** | 网格中区格的默认背景色。未单独设色的区格会使用此颜色 |
| **backgroundImage** | 地图整体的背景图片 URL，铺在网格下方 |
| **defaultBackgroundImage** | 默认的区格场景背景图。未单独配置背景图的区格会使用此图 |

## 网格编辑

地图编辑器提供以下网格工具：

| 工具 | 说明 |
|------|------|
| **选择（Select）** | 点击区格进行选中，查看和编辑其属性 |
| **空白（Blank）** | 点击已有区格将其清除为空白格 |
| **区格（Cell）** | 点击空白格创建新区格 |
| **预设（Presets）** | 使用装饰预设绘制非交互的装饰格（见「装饰预设」一节） |

### 行列操作

- 四方向添加/删除行列，按钮分组为「行: [↑+] [↑-] [↓+] [↓-]」「列: [←+] [←-] [→+] [→-]」
- 添加：在对应方向插入空白行/列，已有区格的位置自动调整
- 删除：移除对应方向最外侧的行/列，该行/列上的区格一并删除

## 区格（Cell）

区格是地图中可交互的场所单位。选中一个区格后，可以编辑以下属性：

| 属性 | 说明 |
|------|------|
| **id** | 区格的唯一标识符，用于连接和脚本引用 |
| **row / col** | 区格在网格中的行列位置（自动设定） |
| **name** | 区格的显示名称，如「酒馆大厅」「市场广场」 |
| **description** | 区格简介（可选） |
| **tags** | 区格标签，可用于行动条件判断（如 `indoor`、`shop`） |
| **backgroundImage** | 该区格的场景背景图。当玩家位于此区格时，游戏界面会显示这张图作为场景背景 |

## 连接

连接定义了区格之间的通行关系。每个区格可以配置多条连接，指向其他区格。

| 属性 | 说明 |
|------|------|
| **targetCell** | 目标区格 ID（同一张地图内） |
| **targetMap** | 目标地图 ID。留空表示同一张地图内的连接；填写后变为**跨地图连接** |
| **travelTime** | 移动耗时，单位为分钟。角色从当前区格移动到目标区格需要消耗的游戏时间 |
| **可感知** | 勾选后 NPC 可透过此连接感知到对面的角色（默认勾选） |
| **仅感知** | 勾选后此连接只用于感知，角色不可通行 |

### 连接的感知与通行

NPC 的行为系统会根据感知范围选择行动目标。默认情况下，NPC 可以感知到通过连接可达的其他角色。

- **取消「可感知」**: 连接会阻断感知传播——NPC 仍然可以通行，但不会「看到」连接另一端的角色。适用于隔音门、楼层间隔等场景。
- **勾选「仅感知」**: 连接只用于感知，角色不可通行。适用于玻璃墙、阳台俯瞰广场等场景——NPC 能看到对面但走不过去。

**默认值**: 同地图内新建的连接默认可通行、可感知；跨地图连接默认可通行、不可感知。

## 单向连接与双向连接

连接本身是**单向**的——从区格 A 连接到区格 B，不代表 B 能回到 A。

编辑器提供了便捷操作：

- **[+单向连接]**：在当前区格创建一条指向目标区格的连接
- **[+双向连接]**：同时在两个区格之间创建互相指向的连接

按钮会自动选择一个尚未连接的区格作为默认目标，可通过下拉框修改。同一区格不允许创建重复的连接（目标下拉框中已连接的区格会被隐藏）。

单向连接适合特殊场景，比如：
- 悬崖：可以跳下去，但不能爬回来
- 暗门：从一侧可以打开，另一侧看不到入口
- 传送阵：单向传送到目标位置

大多数情况下推荐使用双向连接。

## 连接可视化

在地图编辑器中可以开启**连接覆盖层**，直观查看区格之间的连接关系：

| 显示 | 含义 |
|------|------|
| **绿色箭头** | 普通连接（可通行、可感知） |
| **黄色箭头** | 感知被阻断的连接（可通行、不可感知） |
| **蓝色箭头** | 仅感知连接（不可通行、可感知） |

箭头方向表示连接的方向。双向连接会显示两个方向的箭头。

## 装饰预设

装饰预设用于绘制**非交互的装饰格**——它们只有视觉效果（文字和颜色），不是区格，角色不能进入或互动。

典型用途：
- 用颜色块表示墙壁、河流、围栏
- 用文字符号标注地形（如 `~` 表示水，`#` 表示墙）
- 美化地图的视觉效果

### 使用方式

1. 在装饰预设列表中选择或创建预设（设置文字和颜色）
2. 切换到**预设工具**
3. 在网格的空白位置点击绘制

装饰预设的定义保存在扩展包的 `decor_presets.json` 中，可跨地图复用。

## 背景图

地图系统有三个层级的背景图，从低到高优先级依次为：

| 层级 | 属性 | 作用 |
|------|------|------|
| **地图背景** | `backgroundImage` | 铺在整个网格下方，适合大范围的地形底图 |
| **默认场景背景** | `defaultBackgroundImage` | 当玩家进入任意区格时的默认场景图。未配置专属背景图的区格会使用此图 |
| **区格场景背景** | 区格的 `backgroundImage` | 该区格专属的场景背景图。玩家进入该区格时显示，优先级最高 |

使用建议：
- 为地图整体设置一张 `defaultBackgroundImage` 作为通用场景图
- 为重要场所（酒馆内部、神殿等）单独设置区格的 `backgroundImage`
- `backgroundImage`（地图级）适合在编辑器中提供地形参考底图

## 地图合集

一个扩展包可以包含多张地图。在文件层面：

| 文件 | 说明 |
|------|------|
| `map_collection.json` | 地图索引，记录所有地图的 ID 和基础信息 |
| `maps/<mapId>.json` | 每张地图的完整数据，存放在 `maps/` 目录下 |

在编辑器中，你可以通过顶部的地图选择器切换不同地图，也可以创建新地图或删除已有地图。

## 典型用例

### 创建一个小镇地图

假设你要创建一个包含酒馆、市场、住宅区的小镇：

**第一步：规划布局**

```
  [墙] [墙] [墙]
  [墙] [酒馆] [墙]          [市场广场]
  [墙] [门厅] [墙]    ←→    [中央街道]    ←→   [住宅区]
                              ↓
                          [镇门外]
```

**第二步：绘制区格**

1. 在地图编辑器中使用**区格工具**，在网格上点击创建各个场所
2. 为每个区格设置 `id` 和 `name`：
   - `tavern_hall`（酒馆大厅）
   - `tavern_entrance`（酒馆门厅）
   - `market`（市场广场）
   - `main_street`（中央街道）
   - `residential`（住宅区）
   - `town_gate`（镇门外）

**第三步：创建连接**

1. 选中「酒馆大厅」，添加双向连接到「酒馆门厅」
2. 选中「酒馆门厅」，添加双向连接到「中央街道」
3. 选中「中央街道」，添加双向连接到「市场广场」「住宅区」「镇门外」
4. 为酒馆内部的连接启用 `senseBlocked`，使外面的 NPC 看不到酒馆内的情况

**第四步：装饰**

1. 创建装饰预设：墙壁（`#`，深灰色）
2. 使用预设工具在酒馆周围绘制墙壁装饰格

**第五步：设置背景图**

1. 为地图设置 `defaultBackgroundImage`（小镇街景）
2. 为「酒馆大厅」单独设置 `backgroundImage`（酒馆内部场景）
3. 为「市场广场」单独设置 `backgroundImage`（市场场景）

**第六步：保存**

1. 在地图编辑器中点击 **[确定]**，将修改提交到内存（此时修改已生效，但尚未写入磁盘）
2. 点击底部 **[保存变更]**，将所有修改写入扩展包文件

这是所有编辑器通用的两步保存模式：[确定] 提交到内存 → [保存变更] 写入磁盘。
