# 世界侧边栏优化方案

## 对照：Addon 侧边栏 vs 世界侧边栏

| 维度 | Addon 侧边栏 | 世界侧边栏 |
|------|-------------|-----------|
| Header | `Add-on (N)` + `+` 按钮 | 两个按钮 `[新建世界]` `[空]`，创建表单内联展开 |
| 卡片头部 | ToggleSwitch + 名称 + id/version + 展开箭头 | W 图标 + 名称 + addon 数量 + "当前" 标签 + 箭头 |
| 展开面板 | 描述 → 信息行 → 功能按钮（编辑信息/版本管理/删除）→ 版本切换列表 | 查看模式 / 编辑模式 二选一；按钮：切换/编辑信息/删除 |
| 卡片样式 | `borderRadius: 6px`，启用时 `borderLight` | `borderRadius: 3px`，当前世界 accent 边框 |
| 创建流程 | Overlay 弹窗（ID/名称/版本） | 内联表单（名称/ID） |
| 删除 | 未启用时显示 `[删除]`，有确认弹窗 | 始终显示，`confirm()` 原生弹窗 |

---

## 问题与优化点

### 1. 样式不统一

**问题**: 世界卡片 `borderRadius: 3px`、addon 卡片 `6px`；世界用 W 图标占位，addon 用 ToggleSwitch；世界头部按钮用 `[方括号]` 文字，addon 用 `+` 小按钮。两个侧边栏放在一起风格割裂。

**方案**: 统一卡片样式 — `borderRadius: 6px`，去掉 W 图标（信息密度低），header 改为和 addon 一致的 `标题 (数量)` + `+` 按钮布局。

### 2. 创建世界的交互粗糙

**问题**: 点击 `[新建世界]` 后内联展开表单，挤在 header 区域，和 `[空]` 按钮、错误提示混在一起。Save-as 表单也是内联的，两个表单共用状态变量（`newName`/`newId`/`error`），容易冲突。

**方案**: 创建世界改为 Overlay 弹窗（和 addon 的 `CreateAddonModal` 一致），独立状态管理。Save-as 同理。Header 区域只保留标题和按钮，不再承载表单。

### 3. 删除确认用原生 confirm()

**问题**: Addon 侧边栏已用自定义 `ConfirmModal`，世界侧边栏还在用浏览器原生 `confirm()`，体验不一致且无法自定义样式。

**方案**: 改用 `ConfirmModal` 组件（已在 AddonSidebar 中定义，可提取为共享组件，或在 WorldSidebar 中复制同样模式）。

### 4. "空" 按钮语义不清

**问题**: `[空]` 按钮的含义是"卸载当前世界进入空状态"，但文字太简短，新用户无法理解。而且它和 `[新建世界]` 并排，容易误点。

**方案**: 不再作为顶部按钮。改为当前世界卡片内的操作（展开面板中，和删除并排，标签改为 `[卸载]`），只有当前世界才显示。

### 5. 展开面板信息层级不清

**问题**: 展开后查看模式只有纯文本（ID、描述、addon 列表），和编辑模式完全切换。没有类似 addon 侧边栏的 "功能按钮区" + "信息区" 分层。

**方案**: 对齐 addon 侧边栏的面板结构：
- **信息区**: ID、描述（如有）、addon 列表
- **按钮区**: `编辑信息` / `卸载`（当前世界）/ `[删除]`
- **编辑面板**: 点击"编辑信息"后展开（用竖线 `borderLeft` 样式），而非替换整个面板

### 6. 缺少双击/快速切换世界

**问题**: 切换世界需要展开卡片 → 点击 `[切换]`，两步操作。

**方案**: 双击卡片头部直接切换世界（非当前世界时）。保留展开面板中的 `[切换]` 按钮作为备选。

### 7. 世界卡片缺少 addon 摘要

**问题**: 卡片头部只显示 `N addons` 数字，没有具体信息。展开后 addon 列表是 `id@version` 原始格式，难以阅读。

**方案**: 卡片头部保持简洁的数量显示。展开面板中 addon 列表改为使用 addon 名称（如果可获取），否则保持 id，格式改为 Tag 样式小标签。

### 8. 当前世界的视觉区分不够

**问题**: 当前世界只靠一个 "当前" 文字标签和 accent 边框区分，不够醒目。

**方案**: 当前世界卡片 header 背景加微弱的 accent 底色（如 `${T.accent}08`），和 addon 侧边栏的启用态卡片对齐。

## 改动清单

| 序号 | 改动 | 文件 |
|------|------|------|
| 1 | Header: `世界 (N)` + `+` 按钮（弹创建弹窗），移除内联表单和 `[空]` 按钮 | WorldSidebar.tsx |
| 2 | 创建世界改为 Overlay 弹窗 | WorldSidebar.tsx |
| 3 | 卡片样式统一：borderRadius 6px，去掉 W 图标，当前世界加 accent 底色 | WorldSidebar.tsx |
| 4 | 展开面板重构：信息区 + 按钮区（编辑信息 / 卸载 / 删除）+ 竖线编辑面板 | WorldSidebar.tsx |
| 5 | 删除确认改用 ConfirmModal / Overlay 弹窗 | WorldSidebar.tsx |
| 6 | 双击卡片头部快速切换世界 | WorldSidebar.tsx |
| 7 | `[卸载]` 移入当前世界的展开面板按钮区 | WorldSidebar.tsx |

仅改动 `WorldSidebar.tsx` 一个文件。无后端改动。
