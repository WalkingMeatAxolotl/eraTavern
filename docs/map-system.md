# 地图系统

## 1. 概述

地图系统是游戏的空间基础。采用 ERA 风格的网格（Grid）布局，在文本界面中呈现游戏世界的空间结构。

核心设计原则：
- **完全配置化**：所有地图通过 JSON 文件定义，不存在硬编码地图
- **视觉与逻辑分离**：地图的视觉表现（装饰性文字/符号）与游戏逻辑（可移动方格）分开定义
- **装饰性方格无需对象化**：装饰性方格仅存在于视觉层，不参与任何游戏逻辑，不需要成为数据对象
- **加载即编译**：地图加载后，视觉部分编译为静态显示字符串，运行时不再变化
- **跨地图连接**：可移动方格之间的连接可以跨越不同地图

## 2. 层级结构

```
MapCollection (地图集 / 一个游戏世界)
├── Map (地图 / 一个标签页，如 "红魔馆1F与地下室")
│   ├── grid: 二维字符串数组（视觉层，包含所有装饰性文字和可移动方格的显示）
│   └── cells: 可移动方格列表（逻辑层，仅可移动方格）
├── Map ("红魔馆2F与3F")
│   ├── grid: 一个统一的网格（"二楼"和"三楼"在同一 XY 坐标系中）
│   └── cells: [11, 12, 13, 14, 15, 16, 24, 26, 36, ...]
└── Map ("红魔馆-雾之湖西北部")
    └── ...
```

### 层级说明

| 层级 | 说明 | 示例 |
|------|------|------|
| **MapCollection** | 一个完整游戏世界的所有地图 | 整个幻想乡 |
| **Map** | 一个地图标签页，UI 底部显示为可切换的标签 | `[红魔馆2F与3F]` |
| **Cell** | 可移动方格，拥有唯一编号，是游戏逻辑的最小空间单位 | 编号 11 的方格 |

注意：截图中的"二楼"和"三楼"属于**同一个 Map**，共享同一个 XY 坐标系。"二楼""三楼"只是网格中的视觉标注文字，不构成独立的数据层级。

## 3. 视觉层：Grid

### 3.1 定义方式

Grid 是一个二维字符串数组，每个元素代表一个网格方格的显示内容。

```
grid[row][col] = 该位置显示的文字/符号
```

- 装饰性方格：直接在 grid 中写入显示文字（如 `"田"`, `"目"`, `"X"`, `"竹"` 等）
- 可移动方格：在 grid 中写入该方格的编号（如 `"11"`, `"12"`）
- 空位置：使用空字符串 `""` 表示

### 3.2 颜色属性

每个方格支持逐方格定义颜色（RGB）。Grid 中的每个元素可以是以下两种格式：

- **纯字符串**：使用地图默认颜色。`"田"`
- **数组 `[text, color]`**：指定该方格的颜色。`["田", "#8B0000"]`

```
grid[row][col] = "符号"                → 使用 defaultColor
grid[row][col] = ["符号", "#FF0000"]   → 使用指定的 RGB 颜色
```

这样大多数装饰性方格只需写字符串（使用默认色），只有需要特殊颜色的方格才用数组格式，保持配置简洁。

### 3.3 加载后处理

地图 JSON 加载后，grid 被编译为一个**静态显示字符串**，在整个游戏运行期间不会改变。前端渲染时直接使用这个字符串，不需要每帧重新计算。

动态元素（如玩家位置高亮）在静态字符串之上作为**覆盖层**处理。

## 4. 逻辑层：Cell（可移动方格）

只有可移动方格需要成为数据对象，参与游戏逻辑。

### 4.1 Cell 属性

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `number` | 是 | 方格编号，在所属 Map 内唯一 |
| `row` | `number` | 是 | 在 grid 中的行坐标，用于定位视觉位置 |
| `col` | `number` | 是 | 在 grid 中的列坐标，用于定位视觉位置 |
| `name` | `string` | 否 | 位置名称（如 "女仆妖精办公室"） |
| `connections` | `Connection[]` | 是 | 连接的其他方格列表 |

### 4.2 Connection（连接）

连接定义了从一个可移动方格可以移动到哪些其他方格。**连接是单向的**——如果需要双向移动，需要在两端都声明连接。这使得单向通道（如只能从高处跳下）成为可能。

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `targetMap` | `string` | 否 | 目标地图 ID（省略表示同一地图） |
| `targetCell` | `number` | 是 | 目标方格编号 |

连接示例：
- **同地图内移动**：`{ "targetCell": 12 }` → 移动到本地图的 12 号方格
- **跨地图移动**：`{ "targetMap": "scarlet-1f", "targetCell": 14 }` → 移动到"红魔馆1F"地图的 14 号方格

## 5. JSON 配置格式

### 5.1 地图集配置（顶层入口）

```json
{
  "id": "gensokyo",
  "name": "幻想乡",
  "maps": [
    "maps/scarlet-1f.json",
    "maps/scarlet-2f3f.json",
    "maps/misty-lake-nw.json",
    "maps/misty-lake-sw-forest.json"
  ]
}
```

## 6. 运行时行为

### 6.1 加载流程

```
1. 后端读取地图集 JSON
2. 逐个加载 Map JSON 文件
3. 将 grid 编译为静态显示字符串（运行期间不变）
4. 构建 Cell 对象和连接关系图（仅可移动方格）
5. 将显示字符串 + Cell 数据发送给前端
```

### 6.2 移动流程

```
1. 玩家点击/选择一个目标方格编号
2. 前端发送移动请求到后端：{ targetCell: 12 } 或 { targetMap: "scarlet-1f", targetCell: 14 }
3. 后端验证：
   a. 玩家当前方格是否与目标方格有连接
   b. 如果是跨地图移动，加载目标地图
4. 验证通过 → 更新玩家位置，推送状态更新给前端
5. 前端更新显示：移除旧位置高亮，添加新位置高亮
```

### 6.3 前端渲染

- **静态层**：显示编译后的地图字符串（不变）
- **动态覆盖层**：
  - 玩家所在方格编号显示为绿色
  - 可移动方格可交互（点击触发移动）

## 7. 后端职责

| 职责 | 说明 |
|------|------|
| **加载与编译** | 读取地图 JSON，编译 grid 为显示字符串，构建 Cell 连接图 |
| **路径验证** | 验证移动请求：目标方格是否存在、当前方格是否有连接到目标 |
| **位置管理** | 维护所有角色（玩家和 NPC）在地图上的当前位置 |
| **状态推送** | 位置变化时推送更新给前端 |

## 8. 前端职责

| 职责 | 说明 |
|------|------|
| **渲染静态地图** | 显示后端编译好的地图字符串 |
| **渲染动态覆盖** | 在可移动方格位置叠加玩家位置高亮和交互元素 |
| **标签切换** | 显示地图标签栏，处理地图切换 |
| **发送移动请求** | 用户点击可移动方格时，发送移动请求到后端 |
